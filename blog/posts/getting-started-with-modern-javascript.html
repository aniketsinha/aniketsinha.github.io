Getting Started with Modern JavaScript: ES6+ Features Every Developer Should Know

Published on January 15, 2024 | 10 min read

JavaScript has evolved dramatically since its introduction in 1995. The ES6 (ECMAScript 2015) specification and subsequent updates have introduced powerful features that have fundamentally changed how developers write JavaScript code. Whether you're a beginner or an experienced developer, understanding these modern JavaScript features is essential for writing clean, efficient, and maintainable code.

In this comprehensive guide, we'll explore the most important ES6+ features that every JavaScript developer should master. From arrow functions to async/await, we'll cover practical examples and real-world use cases that will help you write better JavaScript code.

1. Arrow Functions

Arrow functions are one of the most popular ES6 features. They provide a more concise syntax for writing functions and have a lexical this binding, which means they inherit this from the enclosing scope.

// Traditional function
function add(a, b) {
    return a + b;
}

// Arrow function
const add = (a, b) => a + b;

// Arrow function with multiple statements
const multiply = (a, b) => {
    const result = a * b;
    return result;
};

// Arrow function with single parameter (no parentheses needed)
const square = x => x * x;

// Arrow function with no parameters
const greet = () => console.log('Hello!');

Arrow functions are particularly useful in array methods and callbacks:

const numbers = [1, 2, 3, 4, 5];

// Using arrow functions with map
const doubled = numbers.map(n => n * 2);

// Using arrow functions with filter
const evens = numbers.filter(n => n % 2 === 0);

// Using arrow functions with reduce
const sum = numbers.reduce((acc, n) => acc + n, 0);

However, it's important to note that arrow functions don't have their own this, arguments, super, or new.target. They're not suitable for methods that need their own this context, such as object methods or constructors.

2. Destructuring Assignment

Destructuring allows you to extract values from arrays or properties from objects into distinct variables. This feature makes code more readable and reduces boilerplate.

Array Destructuring

// Basic array destructuring
const colors = ['red', 'green', 'blue'];
const [first, second, third] = colors;
console.log(first);  // 'red'
console.log(second); // 'green'

// Skipping elements
const [first, , third] = colors;

// Default values
const [first = 'black', second = 'white'] = colors;

// Rest operator
const [first, ...rest] = colors;
console.log(rest); // ['green', 'blue']

Object Destructuring

// Basic object destructuring
const person = {
    name: 'John',
    age: 30,
    city: 'New York'
};

const { name, age, city } = person;
console.log(name); // 'John'

// Renaming variables
const { name: personName, age: personAge } = person;

// Default values
const { name, age, country = 'USA' } = person;

// Nested destructuring
const user = {
    name: 'John',
    address: {
        street: '123 Main St',
        city: 'New York'
    }
};

const { address: { city } } = user;
console.log(city); // 'New York'

Destructuring is also useful in function parameters:

// Function with destructured parameters
function greet({ name, age }) {
    console.log(`${name} is ${age} years old`);
}

greet({ name: 'John', age: 30 });

// With default values
function createUser({ name = 'Anonymous', age = 0, email = '' }) {
    return { name, age, email };
}

3. Template Literals

Template literals provide an elegant way to create strings with embedded expressions. They use backticks (`) instead of quotes and support multi-line strings and string interpolation.

// Basic template literal
const name = 'John';
const greeting = `Hello, ${name}!`;

// Multi-line strings
const message = `
    This is a multi-line
    string in JavaScript.
    No need for \\n or concatenation!
`;

// Expression evaluation
const a = 10;
const b = 20;
const result = `The sum of ${a} and ${b} is ${a + b}`;

// Tagged template literals
function highlight(strings, ...values) {
    return strings.reduce((result, string, i) => {
        return result + string + (values[i] ? `${values[i]}` : '');
    }, '');
}

const name = 'John';
const age = 30;
const output = highlight`Hello, my name is ${name} and I am ${age} years old.`;

4. Spread and Rest Operators

The spread operator (...) allows you to expand arrays, objects, or other iterables into individual elements. The rest operator uses the same syntax but collects remaining elements into an array.

Spread Operator

// Spreading arrays
const arr1 = [1, 2, 3];
const arr2 = [4, 5, 6];
const combined = [...arr1, ...arr2]; // [1, 2, 3, 4, 5, 6]

// Copying arrays
const original = [1, 2, 3];
const copy = [...original];

// Spreading objects (ES2018)
const obj1 = { a: 1, b: 2 };
const obj2 = { c: 3, d: 4 };
const merged = { ...obj1, ...obj2 }; // { a: 1, b: 2, c: 3, d: 4 }

// Function arguments
const numbers = [1, 2, 3, 4, 5];
Math.max(...numbers); // 5

Rest Operator

// Rest in function parameters
function sum(...numbers) {
    return numbers.reduce((acc, n) => acc + n, 0);
}

sum(1, 2, 3, 4, 5); // 15

// Rest in destructuring
const [first, second, ...rest] = [1, 2, 3, 4, 5];
console.log(rest); // [3, 4, 5]

const { name, ...otherProps } = { name: 'John', age: 30, city: 'NYC' };
console.log(otherProps); // { age: 30, city: 'NYC' }

5. Default Parameters

ES6 introduced default parameter values, allowing you to specify default values for function parameters if they're not provided or are undefined.

// Basic default parameters
function greet(name = 'Guest', greeting = 'Hello') {
    return `${greeting}, ${name}!`;
}

greet(); // 'Hello, Guest!'
greet('John'); // 'Hello, John!'
greet('John', 'Hi'); // 'Hi, John!'

// Default parameters with expressions
function createUser(name, createdAt = new Date()) {
    return { name, createdAt };
}

// Default parameters can reference previous parameters
function createRectangle(width = 10, height = width) {
    return { width, height };
}

6. Promises and Async/Await

Promises provide a cleaner way to handle asynchronous operations compared to callbacks. Async/await, introduced in ES2017, makes asynchronous code look and behave more like synchronous code.

Promises

// Creating a promise
const fetchData = () => {
    return new Promise((resolve, reject) => {
        setTimeout(() => {
            const data = { id: 1, name: 'John' };
            resolve(data);
        }, 1000);
    });
};

// Using promises
fetchData()
    .then(data => {
        console.log('Data received:', data);
        return data.id;
    })
    .then(id => console.log('ID:', id))
    .catch(error => console.error('Error:', error));

// Promise.all - wait for all promises
const promise1 = Promise.resolve(1);
const promise2 = Promise.resolve(2);
const promise3 = Promise.resolve(3);

Promise.all([promise1, promise2, promise3])
    .then(values => console.log(values)); // [1, 2, 3]

Async/Await

// Using async/await
async function fetchUserData() {
    try {
        const response = await fetch('https://api.example.com/user');
        const data = await response.json();
        console.log('User data:', data);
        return data;
    } catch (error) {
        console.error('Error fetching user:', error);
        throw error;
    }
}

// Multiple async operations
async function fetchMultipleUsers() {
    const [user1, user2, user3] = await Promise.all([
        fetchUser(1),
        fetchUser(2),
        fetchUser(3)
    ]);
    return [user1, user2, user3];
}

7. Classes

ES6 introduced class syntax, providing a cleaner way to work with constructor functions and prototypes. While JavaScript classes are syntactic sugar over prototypes, they make object-oriented programming more intuitive.

// Basic class
class Person {
    constructor(name, age) {
        this.name = name;
        this.age = age;
    }

    greet() {
        return `Hello, I'm ${this.name}`;
    }

    getAge() {
        return this.age;
    }
}

const person = new Person('John', 30);
console.log(person.greet()); // 'Hello, I'm John'

// Class inheritance
class Student extends Person {
    constructor(name, age, school) {
        super(name, age);
        this.school = school;
    }

    study() {
        return `${this.name} is studying at ${this.school}`;
    }
}

const student = new Student('Jane', 20, 'University');
console.log(student.greet()); // 'Hello, I'm Jane'
console.log(student.study()); // 'Jane is studying at University'

// Static methods
class MathUtils {
    static add(a, b) {
        return a + b;
    }

    static multiply(a, b) {
        return a * b;
    }
}

MathUtils.add(2, 3); // 5

8. Modules (import/export)

ES6 modules provide a standardized way to organize and share code between files. They support both named and default exports.

// math.js - exporting
export const add = (a, b) => a + b;
export const subtract = (a, b) => a - b;
export const multiply = (a, b) => a * b;

// Default export
export default function divide(a, b) {
    return a / b;
}

// app.js - importing
import divide, { add, subtract, multiply } from './math.js';

// Or import everything
import * as math from './math.js';
math.add(2, 3);

// Renaming imports
import { add as sum } from './math.js';

9. Map, Set, WeakMap, and WeakSet

ES6 introduced new data structures that provide better alternatives to objects and arrays in certain scenarios.

Map

// Creating and using Map
const map = new Map();
map.set('name', 'John');
map.set('age', 30);

console.log(map.get('name')); // 'John'
console.log(map.has('age')); // true
console.log(map.size); // 2

// Iterating over Map
for (const [key, value] of map) {
    console.log(`${key}: ${value}`);
}

// Map vs Object
// - Map keys can be any type, Object keys must be strings/symbols
// - Map has a size property
// - Map is iterable
// - Better performance for frequent additions/deletions

Set

// Creating and using Set
const set = new Set([1, 2, 3, 3, 4, 4]);
console.log(set); // Set {1, 2, 3, 4}

set.add(5);
set.delete(2);
console.log(set.has(3)); // true

// Removing duplicates from array
const numbers = [1, 2, 2, 3, 3, 4];
const unique = [...new Set(numbers)]; // [1, 2, 3, 4]

10. Enhanced Object Literals

ES6 enhanced object literals with shorthand property names, computed property names, and method definitions.

// Shorthand property names
const name = 'John';
const age = 30;
const person = { name, age }; // { name: 'John', age: 30 }

// Shorthand method names
const calculator = {
    add(a, b) {
        return a + b;
    },
    subtract(a, b) {
        return a - b;
    }
};

// Computed property names
const prop = 'name';
const obj = {
    [prop]: 'John',
    [`${prop}Length`]: 4
}; // { name: 'John', nameLength: 4 }

Conclusion

Modern JavaScript features have transformed the language, making it more powerful, expressive, and easier to work with. The features we've covered in this guide—arrow functions, destructuring, template literals, spread/rest operators, promises, async/await, classes, modules, and new data structures—are essential tools in every JavaScript developer's toolkit.

As you continue your JavaScript journey, practice using these features in your projects. Start with the basics like arrow functions and template literals, then gradually incorporate more advanced features like async/await and modules. Remember, the goal is to write code that's not just functional, but also readable, maintainable, and efficient.
